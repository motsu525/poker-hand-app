# 1. サービス概要

- ライブポーカーのハンド履歴を簡単に入力できるサービス
- 入力したデータを分析ツールに即連携できる
- 振り返りのハードルを下げ、プレイヤーの上達を支援する

# 2. このアイデアはどこから生まれたか

## 2-1. きっかけとなった体験・感情

- いつ：ライブポーカーをプレイした直後
- 状況：プレイ後にハンドを振り返ろうとしたが、記録が曖昧
- 気持ち：
  - 「あのハンドちゃんと検証したいのに思い出せない」
  - 「メモするの面倒で結局やらない」
  - 「GTOで分析したいのに入力がダルい」

## 2-2. なぜそれが気になったのか

- ポーカーを上達したいという意識が強い
- ロジカルに振り返ることを重視している
- 「改善できるはずの機会を逃している感覚」がストレスだった

→ 成長機会の損失に敏感な性格

# 3. 課題の整理

## 3-1. 表に見えている困りごと

- ハンド履歴を正確に記録するのが面倒
- GTOツールに入力するのが手間

## 3-2. 本当に解決したい課題

- なぜ？ → 入力が面倒だからやらない
- なぜ？ → 手作業だと時間がかかるしミスも多い
- なぜ？ → プレイ直後は疲れていて再現する気力がない

→ 本当に解決したい課題：
「振り返りたいのに、面倒さによって行動できない状態」

# 4. 想定ユーザー

## 4-1. 想定しているユーザー


- ライブポーカーを定期的にプレイ
- セッション後に振り返りをしたい
- 理由：学習意欲があるが、記録が続かない

環境：
- カジノ・アミューズメント施設
- スマホ中心
- 理由：PCを開かずに記録したいニーズがある

## 4-2. 自分とユーザーの距離

→ 自分自身

理由：
- 自分がまさに困っている
- 解像度の高い課題理解がある

## 4-3. 実際に使われる可能性

- 最初のユーザー：自分 + ポーカー仲間
- 利用シーン：プレイ直後 or 帰宅後
- 代替手段：
  - メモ帳
  - 頭で記憶
  - 入力を諦める
- 置き換え理由：
  - 圧倒的に楽なら使う
  - ワンクリックで分析に繋がるなら価値あり

## 4-4. 導入・継続の流れ

- きっかけ：
  - 「振り返りたいけど面倒」
- 初回：
  - ログイン → ハンド入力 → コピー
- 継続理由：
  - 上達実感
  - 記録が蓄積される
  - 後で分析できる
